# Case Study — 从交互中发现的非平凡模式

> 数据：`lmarena-ai/leaderboard-dataset`，最新快照 `2026-05-22`，全量 160 万行、14 个 arena、571 个模型、85 个机构、时间跨度 2023-05 ~ 2026-05。
> 下文每个发现都给出**可在系统中复现的交互路径**，强调多视图协调（brushing & linking）与 details-on-demand 如何让结论"被看见"。

---

## 为什么用可视分析，而不是纯统计 / 机器学习

LMArena 的榜单本质是一个**多维、随时间演化、带不确定性的关系型数据立方**：
`(arena × category × model × organization × snapshot) → (rating, 置信区间, votes, rank)`。

- 分析任务是**探索式**的：分析者事先并不知道"该问哪个问题"——是看头部更替？看机构格局？看风格控制的影响？还是看某模型的偏科？单一统计量（如一张相关系数表）只能回答一个被预设好的问题，无法支撑这种"边看边问"的过程。
- 数据带有**强不确定性**（Bradley–Terry 置信区间），名次差异常常落在区间重叠内。纯排名表会诱导读者把"第3 vs 第5"当成确定结论，而可视化用区间长度 + 票数大小把"这个差异是否可信"直接编码进图形。
- 关系是**跨视图**的：一个模型同时属于某机构、活跃于多个类别、在时间轴上移动。机器学习给不出"为什么"，而 brushing & linking 让用户在机构图里点一下，立刻在榜单、演化、详情里看到同一对象的多副面孔——这正是统计/ML 难以替代的**关联探索**价值。

## 发现 1：头部并非稳定霸权，而是高频更替的"前沿竞赛"

**结论**：在 `text / overall` 的 214 个历史快照中，曾登上 **#1 的模型多达 22 个**；2024 下半年起 #1 几乎每隔几周易主，依次在 xAI → OpenAI → DeepSeek → Google → Anthropic 之间传递（如 `o1-preview`(2024-09) → `deepseek-r1`(2025-01) → `grok-3`(2025-03) → `o3`(2025-04) → `gemini-2.5-pro`(2025-06) → `gemini-3-pro`(2025-11) → `claude-opus-4-6`(2026-02)）。头部是一场**持续的拉锯**而非某家长期垄断。

**为什么 non-trivial**：只看"当前榜单"会得到"Anthropic/Google 领先"的静态印象；只有把时间维度展开（bump chart）才能看到领先权的快速轮转，以及新模型"空降高位→被反超→滑落"的生命周期。

**复现路径**：
1. 顶部筛选 `Arena=text`、`Category=overall`，进入 **排名演化** 标签。
2. 观察多条线频繁交叉；**点击右侧图例**只保留 OpenAI / Google / Anthropic 三家系列，交叉更清晰。
3. 在曲线上**横向拖拽刷选** 2025-01 ~ 2025-06，时间窗放大到 R1 / Grok-3 / o3 互相反超的密集区段；**悬停十字准线**读出任一快照当天的完整名次排序。

## 发现 2：评分不确定性几乎完全由票数决定（Pearson ≈ −0.91）

**结论**：在 `text / overall` 中，置信区间宽度与 `log10(votes)` 的皮尔逊相关达到 **−0.912**——票数越多区间越窄。低票模型（~800 票）区间宽达 **30–42** Elo，而高票模型（>10 万票）区间收窄到 **5–7** Elo。这意味着**很多相邻名次的差距完全淹没在不确定性里**，"第8 vs 第12"在统计上往往不可区分。

**为什么 non-trivial**：榜单默认按 rating 硬排序，读者会把名次当成确定结论。把"区间长度（不确定性）"与"气泡大小（票数）"两个视觉通道并置后，"窄区间=可信、宽区间=存疑"一眼可辨——这是排名表给不了的判读。

**复现路径**：
1. **当前榜单** 标签，`Arena=text / Category=overall`，把 `Rank Top` 调到 30。
2. 看"评分与不确定性"图：**横线长度=置信区间**，**气泡大小=票数**。新上榜/小票模型横线明显更长。
3. **悬停**任一模型联动高亮，对比头部大票模型（短横线、大气泡）与尾部小票模型（长横线、小气泡）；横线相互重叠处即"名次差不可信"。

## 发现 3：风格控制（style control）能把名次搬动 50–70 位，暴露"文风红利"

**结论**：对比 `text` 与 `text_style_control`（同一模型、控制回答长度/格式等风格因素后的榜单），同一模型名次可发生剧烈位移：
- `gpt-5.3-chat-latest`：#117 → **#45**（控制风格后**上升 72 位**）
- `gpt-5.5-instant`：#75 → #21（+54）
- 反向：`amazon-nova-experimental-chat` 系列普遍**下跌 50–63 位**（如 #86 → #149）。

说明部分模型的原始高名次有相当部分来自**讨好式文风（更长、更漂亮的排版）**而非内容质量；去除风格因素后，另一些"内容扎实但文风朴素"的模型才浮上来。

**为什么 non-trivial**：这是"两个榜单之间的差异"，任何单榜视图都看不到；必须借助 arena 间的交叉切换 + 同一模型在两视图中的联动定位才能发现。

**复现路径**：
1. **当前榜单**，`Arena=text`，记下某模型名次；用**顶部搜索框**输入该模型名跨视图定位高亮。
2. 把 `Arena` 切到 `text_style_control`（类别仍 `overall`），同一搜索高亮仍在——直接对比两个名次。
3. 或点该模型打开**详情抽屉**，在"类别强弱热力"里对照其在不同 arena 的排名块色差。

---

## 发现 4：机构格局是寡头 + 长尾——少数家垄断最好名次，多数家"有广度无高度"

**结论**：`text / overall` 前 20 名里，**Google(5) + Anthropic(4) + OpenAI(4) = 13/20** 由三家瓜分；前 10 名几乎被 Anthropic / Google 包揽。而在机构散点图（x=模型数量，y=最好名次）上呈现明显的**两类分化**：左上角"少而精"（模型不多但能冲到 #1）与右下角"多而平"（投放大量模型却进不了前列）。

**为什么 non-trivial**：把"广度（unique_models）"与"高度（best_rank）"两个维度正交摆放，才能区分"研发深度"与"机型铺量"两种完全不同的机构策略——这是单一柱状排行榜无法表达的二维结构。

**复现路径**：
1. **机构竞争** 标签，`Arena=text`。散点图 y 轴向上=名次更好，气泡大小=总票数。
2. **拖拽框选**左上角区域（best_rank 靠前的几家）→ 详情抽屉给出选中机构的**聚合对比**（平均最好名次、合计模型、合计票数）。
3. **点击**某个气泡聚焦该机构：organization 下拉同步、抽屉列出其各 arena 表现，并可点模型**下钻**到模型详情；切到"当前榜单"标签会看到该机构模型被持续高亮（跨视图联动）。

## 发现 5：存在显著"偏科"模型——同一模型跨类别名次可差 100+ 位

**结论**：在 `text` 各细分类别上，不少模型名次极不均衡：
- `o1-2024-12-17`：`korean` 最好 **#39**，`industry_business` 最差 **#157**（跨度 118）。
- `ring-flash-2.0`：`spanish` #111 ↔ `russian` #241（跨度 130）。
- 多个早期/小模型在 `japanese` 上意外靠前、却在 `creative_writing / multi_turn` 上垫底。

说明"综合榜名次"会掩盖能力结构——一个 overall 中游的模型可能是某个垂直类别的强者，反之亦然。**选型应看任务对口的类别，而非 overall。**

**为什么 non-trivial**：这是**单个模型内部的跨类别方差**，在任何按 overall 排序的列表里都不可见，必须借助 details-on-demand 把该模型的全部 (arena, category) 记录摊开按名次着色才能识别。

**复现路径**：
1. 任意视图点击某模型（或用搜索框定位）打开**详情抽屉**。
2. 看"**类别强弱热力**"列表：每行一个 (arena/category)，**色块按名次着色**（青=强、红=弱）。偏科模型会出现明显的青红混杂；通才模型则整体同色。
3. 配合票数条判断每个类别结论的样本量是否充分。

## 交互设计如何服务这些发现（多视图协调小结）

| 能力 | 机制 | 支撑的发现 |
| --- | --- | --- |
| 全局 focus 联动 | 任一视图选中 模型/机构 → 跨标签 `.is-focus`/`.is-dim` 高亮 | 3, 4 |
| details-on-demand | 常驻右侧抽屉：模型类别热力 / 机构各 arena 表现 / 框选聚合 | 4, 5 |
| 时间轴 brushing | 演化图 & 活跃度图共享时间窗，联动 KPI | 1 |
| 联动高亮 + 交互图例 | hover 同机构联动淡出；图例点击隐藏/聚焦系列 | 1, 2 |
| 交叉过滤 | 总览点 arena 切换全局；机构散点框选/点击 → 榜单 | 1, 4 |
| 跨视图搜索 | 顶部搜索框输入即在可见图/表定位高亮、可下钻 | 3, 5 |

这些机制让四个视图不再各自独立，而是围绕同一份选择状态相互印证——分析者得以在"总览 → 聚焦 → 细节"之间自由穿梭，把上面 5 个 non-trivial pattern 一一"看出来"。
